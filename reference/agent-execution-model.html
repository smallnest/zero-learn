二、核心接缝:Provider

type Provider interface {                                  // types.go:209
    StreamCompletion(ctx, request) (<-chan StreamEvent, error)
}

唯一方法。循环只讲 StreamEvent,从不点名厂商 —— 这是支持 25+ provider 的原因。

三、工具与权限

type Tool interface {         // tools/types.go:119
    Name() / Description() / Parameters() / Safety() / Run(ctx, args)
}

工具只用 Safety() 声明危险等级,不做执行检查。三值权限(tools/types.go:32):

值含义例
PermissionAllow直接放行读文件、grep、glob
PermissionPrompt需问用户写文件、跑 shell
PermissionDeny默认拒绝—

可选 ArgsPermissioner:权限随参数变化(loop.go:1665),故 ls 放行、rm -rf 询问。
