zero exec — 一次性执行流程

runExec：参数解析 → 离线工具注册 → 配置解析 → 沙箱与 Provider → agent.Run 单轮循环 → 退出

离线阶段（provider 之前）

  • • parseExecArgs → applyExecMode 展开模式预置
  • • 工作区解析 + 可选 --worktree 隔离副本
  • • 注册核心/escalate/specialist/MCP/plugin 工具
  • • --list-tools 在此短路，全程不启动 provider

装配阶段

  • • config.Resolve 叠加 --model/--max-turns
  • • 无 provider → 直接 provider_error 退出（与 TUI 不同）
  • • NewScope + 重注册 CoreToolsScoped
  • • buildExecSandboxEngine + buildProvider

执行与退出

  • • agent.Run 单次运行，RequireCompletionSignal
  • • 回调驱动 execEventWriter + sessionRecorder
  • • 输出：text / -o json / stream-json
  • • 中断=interrupted · 失败=provider_error · 成功=0
