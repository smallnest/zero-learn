Zero 架构 · 课程目录

读一个真实的终端 coding agent,一课一个能自己复刻的零件

这门课基于 github.com/Gitlawb/zero 的一手源码,把一个现代终端 coding agent 拆成二十个可独立带走的架构决定。 每一课只讲一件事,配三道回忆测验和一份「接下来该读的源码」清单。建议按序读——后面的课常回指前面的课。

Mission:通过读真实源码理解一个现代终端 coding agent 的架构,直到能自己画出一个可比的 agent 骨架。

一 · 核心循环与上下文

agent 怎么跑起来、每一步怎么被守住、上下文怎么不爆

  • 01Agent 主循环流式调模型 → 收工具调用 → 执行 → 回灌,一个 for 循环就是 agent 的心跳
  • 02executeToolCall:工具执行前的层层关卡一次工具调用要穿过哪几道闸门才准跑
  • 03上下文压缩:守住模型窗口对话变长时怎么塌缩历史又保住关键结构
  • 04护栏:让循环不至于失控硬停 vs 一次性提醒,两类护栏各管什么
  • 05系统提示是怎么拼出来的预算受限的分段组合,一次运行只拼一次以吃 prompt 缓存

二 · 交互与持久化

阻塞式 agent 怎么接进 TUI、会话怎么落盘与时间旅行

  • 06TUI:把流式 agent 接进 Elm 事件循环回调 → tea.Msg,权限/追问用 channel 会合
  • 07会话持久化与时间旅行事件日志作真相 + 内容寻址快照,/rewind 原子回溯
  • 08会话回放:把事件日志重新塌缩成一条磁带RehydrateEvents 折叠最后一次压缩,决策与执行分离

三 · Provider 与流

一个接口喂饱各家模型、共享地基、流怎么被抽干、什么时候敢重发

  • 09Provider 适配层:一个接口喂饱 25+ 家模型封闭事件词表作防腐边界,双向翻译——最该复刻的一条边界
  • 10providerio 共享层:所有适配器脚下的地基非幂等的 completion 绝不重放,只重试「服务端证明没收下」的错
  • 11CollectStreamWithOptions:把事件流抽干成一次结果四个出口全走一个 flush 点,边累积边转发回调
  • 12loop.go 的重试路径一条铁律:没吐出正文才敢把一整轮再发一遍

四 · 沙箱安全

放行/追问/拒绝怎么判、判完怎么把命令真正关进笼子

  • 13沙箱引擎:一条有序判定链顺序即安全语义,allow grant 永远盖不过网络/毁灭性拦截
  • 14从判定到笼子:命令怎么被原生沙箱包裹计划/执行分离,Wrapped 字段绝不撒谎;allowlist 靠实证生长

五 · 会话拓扑

一条会话怎么分叉、怎么挂子会话、子任务进度怎么回流

  • 15会话分叉 vs 子会话Fork 复制历史、Child 挂双向链接,事件溯源就是那张 join 表
  • 16specialist 进度回流一条实时管道 + 一条持久账本,喂同一张卡片

六 · 工具系统与扩展

工具怎么注册分发、外部能力怎么接进来、用户怎么注入可复用指令

  • 17工具注册与分发5 方法小接口 + 一排可选能力,穿一条唯一咽喉同时拿下可扩展与安全
  • 18MCP 接入:外部进程工具变注册表一等公民一个适配器抹平远程/本地,Safety 由宿主强加不容自评
  • 19Skill 系统:磁盘上一叠 SKILL.md做成只读工具由模型按需取用,不预灌进 prompt
  • 20Skill vs 用户命令同是 markdown,模型按需取 vs 人敲了就展开,喂进循环的两端
